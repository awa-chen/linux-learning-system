# 路径是什么（第四课）

## 讲解

"路径"就是文件在目录树里的住址。Linux 用 `/` 把一层层目录串起来，比如 `/home/guest/hello.txt`。

- **绝对路径**：从根目录 `/` 写起，写全住址。无论你在哪都指向同一个文件。例：`/home/guest/notes.txt`。
- **相对路径**：从你当前所在目录（pwd）算起。你在 `/home/guest` 时，写 `notes.txt` 就是指当前目录下的它。
- **`.`** 表示"当前目录"，**`..`** 表示"上层目录"。所以 `cd ..` 就是上楼。
- **`~`** 是你的家目录（这里是 `/home/guest`），`cd ~` 永远能回家。

什么时候用哪个？在脚本里、给别人指路时用绝对路径最稳；自己操作图省事用相对路径。

## 动手实验

1. 用绝对路径直接看文件（无论在哪都能看）：
   ```
   cat /home/guest/hello.txt
   ```
2. 用相对路径看当前目录的文件：
   ```
   cd ~
   cat hello.txt
   ```
3. 用 `..` 从家目录走到 `/home` 再回来看：
   ```
   pwd
   cd ..
   pwd
   cd /home/guest
   ```
4. 进入 projects 后，用相对路径 `..` 回到上一级：
   ```
   cd projects
   cd ..
   pwd
   ```
5. 用 `~` 一步回家，再用绝对路径创建嵌套目录：
   ```
   cd ~
   mkdir -p work/draft
   ls work
   ```

## 常见错误

- **"没有那个文件或目录"但文件明明在**：你可能站错目录了。先用 `pwd` 看身高，再用 `ls` 看脚下有什么，再决定用绝对还是相对路径。
- **把相对路径当绝对路径**：相对路径不以 `/` 开头。开头有 `/` 才是从根目录算。
- **`~` 不展开**：在双引号里 `~` 不会被当成家目录。需要展开就别加引号，或用 `$HOME`。

## 小结

1. `/` 开头是绝对路径，稳；无 `/` 开头是相对路径，看当前位置。
2. `.` 当前、`..` 上层、`~` 家目录。
3. 进阶：下一课学 `rwx` 权限，你会明白"路径对"也可能"权限不够"打不开。
