# 函数、参数与返回值

## 讲解

函数让脚本从"流水账"变成"可复用模块"。Bash 函数定义极简：`func_name() { ... }` 或 `function func_name { ... }`。函数内部**复用位置参数那一套**：`$1 $2 $@` 指的是"调用函数时传的参"，而不是脚本的参数——这点容易和脚本顶层搞混。

返回值有两个层面，务必分清：
- **`return N`**：只能返回 0~255 的整数，用作"退出码/状态"，靠 `$?` 读取。它**不是**用来"返回字符串结果"的。
- 想"吐出"数据，靠 `echo` 打印、调用处用 `$(函数名)` 捕获，例如 `result=$(my_func)`。

这套约定（`return` 表状态、`echo` 表数据）是写清晰脚本的关键。

## 动手实验

1. 定义并调用函数：
   ```bash
   greet() {
     echo "你好, $1"
   }
   greet "Alice"
   ```
   预期：输出 `你好, Alice`。`$1` 是调用时传入的 "Alice"。

2. return 返回状态（用 `$?` 读）：
   ```bash
   is_even() {
     [ $(( $1 % 2 )) -eq 0 ] && return 0 || return 1
   }
   is_even 4
   echo "退出码: $?"      # 0
   is_even 3
   echo "退出码: $?"      # 1
   ```

3. echo 返回数据（用 `$()` 捕获）：
   ```bash
   add() {
     echo $(( $1 + $2 ))
   }
   sum=$(add 3 5)
   echo "3+5=$sum"        # 3+5=8
   ```

4. 函数内用 `local` 声明局部变量（避免污染全局）：
   ```bash
   calc() {
     local tmp=$1
     tmp=$(( tmp * 2 ))
     echo $tmp
   }
   ```

## 常见错误

- **想用 `return` 返回字符串**：`return "ok"` 会报错（return 只接受数字）。返回文本请用 `echo`，调用处 `$(...)` 接住。
- **函数内外 `$1` 混淆**：函数里的 `$1` 是函数参数，不是脚本参数。若需脚本参数，进函数前先存到命名变量或 `$@` 透传。
- **忘了 `local` 导致变量泄露**：函数里直接 `x=1` 会改全局 `x`，多个函数互相干扰。局部变量务必 `local x=1`。
- **`$?` 被提前覆盖**：`echo "退出码: $?"` 之前的 `echo` 本身会改写 `$?`。要保留状态，先 `ret=$?` 存起来再打印。

## 小结

1. 函数用 `name() { ... }`，内部 `$1 $@` 指函数自己的参数。
2. `return N` 表状态（0~255，用 `$?` 读）；要返回数据用 `echo`，调用处 `$(...)` 捕获。
3. `local` 让变量不出函数，避免全局污染——写多函数脚本的纪律。
