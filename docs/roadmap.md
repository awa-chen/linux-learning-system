# Penguin Path 能力总纲路线图（roadmap）

> **总纲定位**：Penguin Path 是一条从「不敢碰终端的小白」到「能设计高可用架构与 SRE 体系的高级工程师」的 12 级递进式 Linux 成长路线，每一级都对应一套可验证的实战能力，而非纸上谈兵。

---

## 三、成长路径与里程碑

### 阶段一：入门（L1~L3）—— 建立肌肉记忆
- **里程碑 M1（完成 L3）**：能用终端完成 90% 的日常文件与文本操作，不再依赖图形界面找文件。
- 标志：看到报错会先 `cat` 日志、`grep` 关键字、`find` 定位。

### 阶段二：初级（L4~L6）—— 会管一台机器
- **里程碑 M2（完成 L6）**：能独立**运维单台服务器**——建用户、管进程、配包源、远程 SSH 上去排错、看懂网络连接。
- 标志：收到「这台机器怎么这么慢/连不上」的工单，你能独立定位到用户、进程或网络层。

### 阶段三：中级（L7~L9）—— 能交付生产系统（交付三角）
- **里程碑 M3（完成 L7）**：能把一个 Web 服务 + 数据库 + 容器**真正跑起来并对外可用**（Nginx 反向代理、Docker 部署、MySQL 可恢复）。
- **里程碑 M4（完成 L8）**：能把 L7 的手工操作**沉淀为可复用的 Ansible Playbook/Roles**，一键在多台机器上重复部署，告别「手工点点点」。
- **里程碑 M5（完成 L9）**：能**搭建监控体系**——指标(Prometheus)、看板(Grafana)、告警(Alertmanager)、日志(集中式)，系统出问题你比用户先知道。
- 标志：从「人肉运维」升级为「用代码和仪表盘运维」。

### 阶段四：高级（L10~L12）—— 能守住与能设计
- **里程碑 M6（完成 L10/L11）**：能**加固系统安全**、能**用数据定位性能瓶颈**，而不是靠猜。
- **里程碑 M7（完成 L12）**：能做**架构与 SRE 方案**——设计高可用拓扑、定义 SLO/Error Budget、组织故障演练(混沌工程)、写容量规划与事故复盘。这是路线终点，也是「高级工程师」与「资深/架构师」的分水岭。

### 一句话路径
> L1~L3 学会用终端 → L4~L6 学会管一台机 → L7 把服务跑起来 → L8 把它批量化 → L9 把它看住 → L10/L11 守住安全与性能 → L12 设计架构与 SRE 体系。
