# Bash 变量、引号与数组

## 讲解

变量是脚本的"记忆"。Bash 里赋值**等号两边不能有空格**（`x=1` 对，`x = 1` 会被当成命令），引用时加 `$`（`echo $x`）。新手 90% 的坑都出在**引号**上：单引号 `'...'` 里的一切都是字面量，双引号 `"..."` 里 `$变量` 和 `` `命令` `` 会被展开。理解这一点，才能避免文件名带空格时脚本炸掉。

除了标量，Bash 还有**数组**（`arr=(a b c)`），以及特殊的"位置参数"：`$0` 是脚本名、`$1` `$2`…是传入的参数、`$@` 是所有参数（保留空格）。这些在写函数/可复用脚本时天天用。

## 动手实验

1. 变量赋值与引用：
   ```bash
   name="Penguin"
   echo "你好, $name"
   echo '你好, $name'     # 单引号不展开
   ```
   预期：第一行输出 `你好, Penguin`；第二行原样输出 `你好, $name`。

2. 命令替换（两种写法）：
   ```bash
   today=$(date +%F)
   files=`ls`            # 老写法，不推荐
   echo "今天是 $today"
   ```

3. 位置参数：
   ```bash
   cat > hello.sh <<'EOF'
   #!/bin/bash
   echo "脚本名: $0"
   echo "第1个参数: $1"
   echo "全部参数: $@"
   EOF
   bash hello.sh alice bob
   ```
   预期：依次打印脚本名、`alice`、`alice bob`。

4. 数组：
   ```bash
   fruits=(apple banana cherry)
   echo "${fruits[0]}"      # apple
   echo "${fruits[@]}"      # apple banana cherry
   echo "${#fruits[@]}"     # 数组长度 3
   ```

## 常见错误

- **`x = 1` 报错 command not found**：等号两侧有空格！正确是 `x=1`。这是最高频错误。
- **变量含空格不加引号**：`rm -rf "$dir"` 若 `$dir` 是 `my dir` 且没引号，会被当成两个参数。引用变量一律加双引号。
- **单双引号用反**：想要展开变量却用了单引号，结果输出字面 `$name`。需要展开用双引号。
- **`$@` 与 `$*` 混用**：`"$@"` 保留每个参数的独立性（推荐），`"$*"` 把所有参数拼成一个字符串。循环遍历参数用 `"$@"`。

## 小结

1. 赋值 `x=1`（无空格），引用加 `$`；单引号字面量、双引号可展开。
2. 命令替换用 `$(...)`，比反引号更安全可读。
3. 位置参数 `$1 $@` 与数组 `("a" "b")` 是脚本复用的基石，引用时记得双引号。
