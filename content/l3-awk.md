# awk 字段提取（第九课）

## 讲解

`awk` 是文本处理三剑客里最"表格化"的一个。它把每行按空白（或你指定的分隔符）切成若干**字段**，然后你可以挑字段、做计算、排版。

基础句式：

```
awk '{print $1}' 文件
```

- `$1` 是第一列，`$2` 第二列……`$0` 是整个原始行，`NF` 是这一行有多少字段。
- 默认按**空白**（空格/制表符）分列。用 `-F` 指定别的列分隔符，如 `-F:` 按冒号分（适合 `/etc/passwd`）。
- `print` 后面可以组合多个字段，用空格或逗号隔开。

awk 特别适合处理"每行结构一致"的日志、表格数据。本模拟器支持 `{print $N}`、`$0`、`NF` 及 `-F` 分隔符。

## 动手实验

1. 看 todo.txt 第一列（序号）：
   ```
   awk '{print $1}' todo.txt
   ```
   预期：`1.` `2.` `3.` `4.`
2. 看 hello.txt 第一列（默认按词分列）：
   ```
   awk '{print $1}' hello.txt
   ```
3. 打印整行和字段数（NF）：
   ```
   awk '{print NF, $0}' notes.txt
   ```
4. 指定分隔符：用 `:` 切 hello.txt 的某一行体会（这里演示 -F 用法，对普通文本也成立）：
   ```
   echo "name:Linux:2026" | awk -F: '{print $2}'
   ```
   预期：`Linux`
5. 组合字段，自定义输出格式：
   ```
   awk '{print "第" $1 " 项"}' todo.txt
   ```
   预期：`第1. 项` 等
6. 管道组合：ls -l 的最后一列是文件名：
   ```
   ls -l | awk '{print $NF}'
   ```

## 常见错误

- **字段序号从 1 开始**：`$0` 才是整行，`$1` 是第一列，没有 `$0` 之外的"第0列"概念。
- **分列不对**：默认按空白分。如果你的数据用逗号/冒号分隔，必须加 `-F,` 或 `-F:` 才能正确分列。
- **只 print 第一个匹配**：awk 默认对每一行都执行，不必循环；写错成 `print $1` 只在外层会只输出一次——确保 `{print ...}` 大括号包住动作。

## 小结

1. `awk '{print $1}'` 取第一列，`$0` 整行，`NF` 字段数，`-F` 指定分隔符。
2. awk 按行处理、按列提取，适合结构化文本/日志。
3. 进阶：三剑客 grep/sed/awk 至此集齐，L4 起进入系统管理实战。
